在用了 DELL 的 Inspiron 560S 近5 年之后，实在受不了各种死机，重启，一直打算收一个 macmini 用来做台式机替代，可是 Apple 公司不给力，一直不好好升级硬件，无奈，我把本来打算转让的旧 Macbook 拿出来替换了一块 SSD 硬盘当台式机用了。旧的 560S 也并不是一无是处，尤其是内存被我升到了 6GB 以后。于是就想拿他做家庭 NAS。

## 前言

这台 DELL Inspiron 560S 是 2011 年的时候买的，当时公司的唯一一台台式机已经老旧的连 xp 都运行缓慢了，经常黑屏，死机，实在影响工作效率，而且旧的笔记本也几乎没有办法正常使用了，于是就去京东上买了这么一台。用了不久屏幕就有一个角落亮起来了，不是亮点，是亮块。鉴于不怎么影响办公使用，也就没有深究，一直这么用下来了。

后来就是无缘无故的死机，重启。上面运行着公司的 Vtiger CRM，有时候在外面想要查一下里面的联系人资料都很受影响，于是检查了硬件，加了内存，却依然如故。

业务收缩后变为 SOHO，在仅有一台电脑的情况下，实在受不了各种状况，于是尝试安装其他版本的操作系统，CentOS 试过，软件方面实在没有办法兼容，后来发现了 Deepin Linux，起初倒是很好，还装上了 VMWare 的 linux 版，里面倒是能跑 windows，不过后来也是无故的蓝屏，死机。想要安装 Mac OS，看了看硬件规格，发现 CPU，显卡都不能很好的兼容。最老的 Macbook 升级了内存和 SSD 硬盘后，一直卖不出去，于是拿来自己用，买了火线转接 VGA 的接线后权当台式机用了，有了 SSD 后速度倒是可以接受了。

于是一直想装 esxi 的心躁动起来了。

## 安装 esxi 6.0

我尝试过用 U 盘，好几次都失败了。。。。。于是，我想到了早已被我束之高阁的 DVD 盘片，虽然 CD 也够了，但我买 CD 岂不是更浪费，于是拿来刻盘，安装。

装到还没有一半就进行不下去了。到网上各种搜索，原来是网卡问题。没有网卡驱动，于是通过搜索找到了适用的版本，并照着教程，下载软件，把驱动装进了镜像中，重新刻盘，安装，一次搞定。

如果大家碰到了和我一样的问题，那么我就做一回搬运工，照着[这篇文章](http://opoo.org/2015/install-vmware-esxi-6.0-with-88se9230-and-rtl8111e/)下载相关工具并制作启动 ISO 即可。

## 管理 esxi 6.0

系统默认是通过 IP 地址访问，安装好之后的页面会给出地址。网页登录后即可进行管理，我是进入后修改成了 192.168.1.X0 这样的固定内网地址，方便记忆和使用。

在网页界面里可以很方便的创建虚拟机，管理 SSH，管理存储空间。都是可视化操作，大家进去一看就能明白。

最好还是安装 Vsphere Client，不过鉴于没有 MAC 版的，我也只是偶尔进 win7 的虚拟机管理一下，基本上就是设置下虚拟机的随系统启动，别的都可以在网页端完成。

## 黑群晖

#### 安装
有一篇让我受益至今的文章，这里一定要给大家分享出来。[没NAS也能体验群晖：VMware虚拟机安装DSM详细安装步骤](http://post.smzdm.com/p/350104/)

当然了，这是我之前在本地使用虚拟机的时候参考的文章。到了 esxi 这里虽然也受用，但是我在使用的时候发现在这一步会卡住，重启总是失败。

![dsm](https://i349.photobucket.com/albums/q365/Athen_Ai/blog/dsm-cdrom.jpg)

经过多处搜索和尝试发现需要用`cdrom, cdrom, cdrom`来替代，我也不知道为什么，可能硬件位置编号不同吧。这样每次更新或重启时黑群就能正常启动了。

此方法暂时适用 5.x 系统，我之前一不小心升级到了 6.0 然后整个系统就废掉了，还好没放什么重要文件在里面，否则找回来都困难，记得每次装好要开启 SSH，一旦出问题就可以救命。6.0 系统的启动 iso 一直没有出来，所以这种方法被否了。

经过长达一周左右的搜索，终于找到了 6.0 的，而且适用 esxi 系统的操作和设置简单到发指！！将相应的文件上传到 esxi 的 datastore 里面然后创建虚拟机，选择导入 ova 文件即可。不用折腾各种引导，不用修改任何文件。启动成功后新建一个 vmdk 的磁盘并且挂载即可。以后所有的文件就都在这个 vmdk 文件里了。备份和恢复都异常简单。

这里我也搬运一下。[在虚拟机Vmware或ESXI中安装Synology5.2、6.0（黑群晖 DSM）](http://blog.sina.com.cn/s/blog_89c4d77a0102vz63.html)

#### 我的备份方案

这里我做了两个 ova 导入，一个叫 Synology，另一个叫 Synology Backup。创建两个大小一样的 vmdk 文件，全都挂载到第一个上，两个系统设置为一致的（这里似乎略微费点时间），创建文件夹以后，在第二个 vmdk 里创建同名的，系统内设置自动同步文件夹即可。如果一个磁盘挂掉，可以把 vmdk 查下来放在 backup 系统里挂载使用，如果挂载不能使用，那就挂载第二个 vmdk，第二个是一定可以使用的，因为里面只有文件，没有任何设置需要读取。

#### 外网访问

作为云存储，如果只能局域网使用，那效果会大打折扣。所以我们需要外网访问，在国内没有固定 IP 的情况下 DDNS 就是必须的了。这其中，花生壳又是做的最好的。

有钱的可以买个花生壳的棒棒，插上就好使。

想省钱如果就需要早下手拿下免费的域名。XXX.eicp.net 或其他的 XXX.net 的二级域名。有了就好办了。不用使用其中的花生壳应用。直接在系统层面设置即可。

使用系统的 EZ Internet，按照向导提示一步一步的设置即可。所有已经安装的应用需要用到的端口都会自动帮你映射好。它能自动识别路由器，并且将设置写入路由器内。如果没识别你的路由器，那么你可以联系运营商更换或者自己淘宝几十块钱买一个。不过，绝大多数中兴和华为的都能很好的识别。

设置好之后你就可以在外网用二级域名访问了。

#### 美观

在开通了 web 功能后，所有的功能都不需要每次输入 域名：5000来进入系统后再使用了。可以直接输入域名然后显示一个界面，里面设置好相应的链接，点连接即可直接进入，简单粗暴，我很喜欢。暂时找不到地址了，找到后我再贴上来把。

#### 数据安全

安全的是数据，而不是硬盘。

除了做 vmdk 的备份之外。我还做了云同步，利用黑群晖里的 Cloud Sync 与 onedrive 同步照片和文档。本地电脑安装 Cloud Station 将本地的文件夹与黑群晖里的文件夹进行同步，如果有 google drive 账号并且可以自由翻墙的话，还可以再做一层 google drive 的备份。空间不够？那是你不会找资源，onedrive 的 1TB 和 google drive 的无限空间都等着你去发现。

## 扩容

双十一的时候京东上下手了一块 2TB 的希捷硬盘，我买的监控型的，需要他 24 小时不间断运行。买来后拆机箱，把里面预留给笔记本硬盘的地方拿来放了原装的 500GB 硬盘，原来的两个硬盘位装入新买的这块硬盘。以后如果再买，可以装入第二个硬盘位，再不够就只能拆光驱了。

装好后进入 esxi 的网页管理界面，创建新的 data store 或者将新硬盘为原 data store 扩容，两种方法孰优孰劣各位根据自身情况评估吧。

## 网盘对比及总结

现在各种网盘找个各种借口停止服务，以前的 115 已经各种花样收费作死，亚马逊 S3 等存储价格一路走低，115 竟然反其道行之，逐年加价不加量，服务也各种阉割，本人早已弃用，型号当年花出去的钱都靠枫币赚回来了。在这里祝他早日倒闭。

360 一直用的都不多，也没感觉人家想要做多好，只是每天顺手签个到赚点空间。现在也停止服务了，东西删了也好。

百度云用户一直都很多，只是安全性堪忧，各种塞文件，删文件，改文件，到底是我的文件还是你的文件？我把钱放在银行，钱就不是我的了？

国内其他的不提也罢。

国外的在这里就推荐 dropbox 和 Mega，dropbox 胜在 API 丰富， Mega 容量大，安全性好，速度偶尔会比较吃力。我最推荐 Mega，名气够响，国内用的少。50GB 的空间够用，尤其是桌面客户端，同步文件特别方便。需要使用的文件简直就是随用随取啊~ dropbox 被墙，否则绝对是第一选择，虽然空间略小，少放点东西又死不了人。

最后还是回到原话题，东西还是自我掌握才最安全，最放心。旧物利用不浪费，加个硬盘才 400，哪有比这个还便宜的？ 2TB 哦，剩下就是电费了~ 百度云一年多少钱？各位看官自己衡量吧。
